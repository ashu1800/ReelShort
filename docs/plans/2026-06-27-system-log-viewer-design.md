# System Log Viewer Design

## 背景

后台已经具备审计日志和运行诊断，但缺少面向单机运维的应用日志查看入口。管理员目前只能通过服务器文件系统排查后端运行问题，不符合架构文档中“系统日志查看”的后台运营能力要求。

## 目标

新增后台系统日志查看能力，让具备权限的管理员可以在 Web 后台读取后端应用日志的最近内容。该能力只服务于排障和运行观察，不作为任意文件浏览器。

## 方案选择

推荐方案：Spring Boot 提供只读 Admin API，限制在配置的日志根目录内读取固定日志文件；Vue 后台新增系统日志页面。该方案和现有 `/api/admin/system/runtime` 模式一致，权限、统一响应、前端会话和错误处理都能复用。

备选方案一：只在部署文档中说明 SSH 查看日志。实现成本低，但后台能力缺失，运营人员仍需服务器权限。

备选方案二：引入完整日志平台或 ELK。能力更强，但和当前长期单机部署定位不匹配，运维成本过高。

## 后端设计

新增 `backend/system/logs` 子模块：

- `SystemLogProperties`：绑定 `reelshort.system.logs.root`，默认 `logs`；绑定 `max-lines`，默认 `500`；绑定 `max-bytes`，默认 `1048576`。
- `SystemLogService`：枚举日志根目录下普通 `.log` 文件，读取选中文件尾部最近 N 行。
- `SystemLogController`：暴露 `GET /api/admin/system/logs`，需要 `SYSTEM_LOG_READ` 权限。
- `SystemLogResponse`：返回可选文件列表、当前文件、行数、是否截断、更新时间和日志行。

安全边界：

- 不接受绝对路径作为文件名。
- 拒绝 `..`、路径分隔符和非 `.log` 文件。
- 对日志根目录和候选文件做 normalize/canonical 边界校验。
- 日志目录不存在时返回空文件列表和空行，不抛出异常。
- `lines` 参数最小为 `1`，最大受配置限制。
- 单次只读取文件尾部最多 `max-bytes` 字节，避免大日志文件造成接口内存压力。

## 前端设计

后台 Web 新增“系统日志”菜单和页面：

- 页面路径：`/system-logs`。
- 默认加载可用日志文件和最近 200 行。
- 支持选择文件、调整行数、刷新。
- 使用等宽文本区展示日志行，避免复杂格式解析。

## 错误处理

- 未登录返回现有 `401`。
- 缺少权限返回现有 `403 forbidden`。
- 非法文件名或非法行数返回 `400`。
- 文件在枚举后被删除时返回空行并刷新文件列表，不泄漏服务器路径。

## 测试

后端测试覆盖：

- 无日志目录时返回空列表。
- 可读取允许日志文件的最后 N 行。
- 路径穿越和绝对路径被拒绝。
- 未登录、缺少权限、默认管理员权限引导。

前端通过 TypeScript 构建验证 API 类型、路由和页面接入。
