# Android 认证页商业级重设计

## 目标
把当前“登录/注册混排”的认证页重排成更像商业消费产品的入口页：默认登录，注册为次级入口，用户一眼能看懂当前流程。

## 设计结论
- 默认进入登录模式。
- 注册保留为次级入口，通过明确的模式切换进入。
- 每个模式只保留一个主 CTA，另一个动作降级为文字链接。
- 验证码输入和 `Send code` 只在注册模式出现。
- 保留当前深色品牌风格，但减少同时出现的交互元素数量，提升层级和留白。

## 视觉与交互原则
- 认证页只突出一条主线，避免登录和注册两个同级按钮同时争夺注意力。
- 标题收紧，品牌区保留，但不抢占表单视觉重心。
- 表单块使用稳定间距和清晰分组，避免“功能堆叠”的感觉。
- 底部弹层继续承载认证，但内部结构更像一个完整流程页，而不是一组零散控件。
- 触控区域保持大面积、明确 pressed feedback，避免小型文字链接成为唯一可点击区域。

## 变更范围
- 只调整 Android App 认证页 UI 和相关状态层。
- 不改后端接口。
- 不改注册验证码规则。
- 不改账号、钱包、提现、交易等商业化功能。

## 预期结果
- 用户打开认证弹层后，能立即看出当前是登录还是注册。
- 默认登录路径更短，注册路径更明确。
- 页面整体更像成熟商业 App 的认证入口，而不是功能测试页。
