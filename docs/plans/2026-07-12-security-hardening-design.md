# 安全审查问题修复设计

## 目标

修复 2026-07-12 全面代码审查确认的七项问题，同时保留本地开发的低配置启动体验，并确保生产环境对缺失安全配置采取 fail-closed 行为。

## 设计决策

### 生产密钥与管理员引导

- 默认配置不再提供管理员密码哈希和支付回调密钥。
- 新增统一的生产安全配置校验：非 `app-dev`、`test` profile 启动时，管理员密码哈希和支付回调密钥为空、为公开开发值或过短均拒绝启动。
- `application-app-dev.properties` 与测试配置显式声明开发凭据；开发便利性不再隐式渗透到生产配置。
- `/api/internal/` 在部署 Nginx 层默认拒绝公网访问；容器内调用不经过公网 Nginx。

### 数据层网络边界

- 生产 `docker-compose.yml` 删除 PostgreSQL 和 Redis 的宿主机端口映射，只保留 Compose 内部网络。
- 新增仅用于本地调试的 Compose override 示例，将端口绑定到 `127.0.0.1`。

### Android 会话安全

- `EncryptedSharedPreferences` 初始化失败时不再回退到明文文件。
- 会话存储改为内存 fail-closed 实现；旧明文会话文件在初始化路径删除，用户需要重新登录。
- 保留纯 Kotlin `FileSessionStore` 作为测试/非 Android 边界实现，但生产组合根不再使用它。

### 内容源出站请求边界

- 所有上游 HTTP 请求禁用自动重定向，并逐跳验证 `http/https`、公开主机和最大重定向次数。
- 拒绝 loopback、私网、链路本地、保留地址及解析到这些地址的域名。
- 使用流式读取和明确的最大响应字节数；超过限制立即失败。
- 用单调时钟实现整个请求链的总 deadline，现有 connect/read timeout 仍作为单次 I/O 边界。

### 备份安全

- 配置备份默认不再复制 `.env`；必须显式启用。
- 启用配置备份时，Windows 使用 DPAPI 加密为当前用户可解密的文件，并收紧备份目录 ACL；恢复脚本兼容解密。
- 文档明确备份介质权限与恢复后的密钥轮换要求。

### 发布基线完整性

- 同时执行 unstaged 与 staged `diff --check`。
- 默认拒绝未跟踪文件；提供显式开关供已知生成物场景使用。
- 脚本回归测试覆盖 staged whitespace 和 untracked 两条失败路径。

## 测试策略

- 后端：配置校验的生产失败、开发允许和弱密钥拒绝测试。
- Android：模拟加密初始化失败，验证不会创建或使用明文会话文件。
- 内容源：重定向私网、超大响应、重定向循环和正常响应契约测试。
- PowerShell：发布基线 staged/untracked 回归测试；备份加密与静态验证。
- 最终运行完整发布基线、`git diff --check` 和安全配置静态检查。
