# Android 观看奖励阶段提示设计

## 背景

后端观看奖励规则已经固定为 `25 / 50 / 75 / 100` 阶段，并由后端保证同一用户、同一分集、同一阶段只发一次。Android 播放页现在已经能同步 Media3 播放进度到 `PlaybackState`，但用户看不到下一次奖励阶段，也不清楚何时点击“上报当前进度”更有意义。

## 目标

- 在播放页展示下一观看奖励阶段提示。
- 当当前进度达到新的阶段但尚未上报时，提示用户可上报领取。
- 当当前进度仍未达到下一阶段时，提示距离下一阶段还差多少。
- 当所有阶段都已上报或当前进度不超过已上报进度时，展示明确状态。
- 不改变后端奖励规则、不自动上报、不在 App 侧发放积分。

## 非目标

- 不读取后台系统配置中的每阶段积分数量。
- 不做自动上报、自动弹窗或强提醒。
- 不修改 `app-core`、后端、积分模型。

## 设计

新增纯 Kotlin UI helper：

- `WatchRewardHint(title, message, actionReady)`
- `watchRewardHint(progressPercent, lastReportedProgressPercent)`

规则：

- 奖励阶段固定：`25 / 50 / 75 / 100`。
- `lastReportedProgressPercent` 表示后端确认后的最高已上报进度。
- 如果当前进度超过已上报进度并达到某个新阶段，提示“可上报领取 X% 阶段奖励”。
- 如果未达到下个阶段，提示“距离 X% 阶段还差 Y%”。
- 如果所有阶段都已上报，提示“本集奖励已完成”。

## UI

播放页在播放元数据和操作按钮之间增加一个 `SurfacePanel`，显示：

- 阶段标题。
- 阶段说明。
- 一个 `MetaPill` 表示“可上报”或“继续观看”。

## 验收标准

- JVM 单测覆盖未开始、未达阶段、达到未上报、已上报阶段、全部完成。
- 播放页显示奖励阶段提示。
- 不自动调用 `reportProgress`。
- APK 能构建并在雷电模拟器启动，无 `FATAL EXCEPTION`。
