# Android 内容状态体验设计

## 背景

App 已经具备深色影院感视觉基础、真实封面图和基础内容浏览能力。当前首页、搜索和详情页在无数据、未搜索、无分集等状态下只显示单行文本，真实接入内容源时不利于用户判断下一步操作。

## 目标

- 将首页空推荐、搜索未执行、搜索无结果、详情无分集区分展示。
- 空态使用更完整的标题、说明和简洁行动提示，保持深色影院风格。
- 保持 `app-core` 数据模型、后端接口和内容源接口不变。
- 文案和状态判断提取为可测试的纯 Kotlin helper，避免只靠人工看 UI。

## 非目标

- 不增加下拉刷新、分页、筛选、排序。
- 不实现真实播放器或内容源新接口。
- 不改变登录、积分、订单等业务流程。

## 设计

- 增加 `ContentEmptyState` 文案模型，包含 `title`、`message`、`actionLabel`。
- 增加纯函数：
  - `homeEmptyState()`
  - `searchEmptyState(query, resultCount)`
  - `detailEmptyState(book, episodeCount)`
- 搜索页根据 `state.searchQuery` 区分：
  - 未搜索：提示输入关键词。
  - 已搜索且无结果：提示换关键词。
  - 有结果：展示结果数量。
- 详情页根据 `selectedBook` 和 `episodes` 区分：
  - 未选择剧集：提示先从首页或搜索选择。
  - 已选择但无分集：提示稍后刷新或换剧。

## 验收标准

- 首页空推荐展示明确标题、说明和“去搜索”行动提示。
- 搜索页首次进入不显示“暂无搜索结果”，而显示搜索引导。
- 搜索无结果时展示包含当前关键词的说明。
- 详情页无分集时展示专门空态，而不是空白列表。
- JVM 单测覆盖上述状态文案。
- APK 能构建并在雷电模拟器启动，无 `FATAL EXCEPTION`。
